01 · The Standard

A universal translator between models and your data

Model Context Protocol is an open standard, started by Anthropic, for connecting AI models to external data without writing a custom integration for every provider.

ZeroCarbon runs the server side of it. Instead of hardcoding what an AI can do with your emissions data, the server publishes a list of tools it supports — query the ledger, calculate a footprint, verify an entry — and the model picks the right one for the question it's been asked.

Every call travels over a JSON-RPC 2.0 bridge, either locally through stdio pipes or remotely over HTTP behind an API key.

02 · What It Does

Three things it hands the model

Once connected, a model doesn't just read your data — it can question it, calculate against it, and log new entries into it.

→ AUDIT

Natural language ledger auditing

Ask a question in plain English and the model turns it into the right database call, then hands back a formatted answer.

"List all Scope 3 travel entries that aren't verified yet."
→ CALCULATE

Instant GHG Protocol calculations

Exposes verified calculation tools for electricity, fuel combustion, air travel, waste, and logistics — using location-specific factors like India's CEA grid emission factors.

electricity_calc · combustion_calc · travel_calc · waste_calc
→ INGEST

Automated data ingestion

Point the model at unstructured text or a file, and it extracts the values, runs the calculation, and records the entry — all from one prompt.

"Log this diesel invoice for the Chennai site."

04 · Architecture

Five layers, one request

Every call — whether it's a chat question or a scheduled scan — passes through the same five layers in order.

1

Clientwhere the request starts

Operators ask questions, write code, or trigger scheduled automation from Claude Desktop, an IDE, or a script.

2

Gatewaywhere it's authenticated

Translates a local pipe or remote HTTP call into a JSON-RPC message and checks the API key signature.

3

MCP Corewhere tools are offered

Publishes the list of available tools and validates that incoming variables match expected categories.

4

Ingestionwhere files become numbers

Monitors a shared mailbox and runs multimodal models to convert invoice images and PDFs into figures.

5

Ledgerwhere it's recorded

Runs the SQL transaction, attaches the source document, and keeps the carbon record's integrity intact.

05 · Connect

Connect in minutes, not sprints

Three ways in, depending on who's connecting and where they're working.

  1. Open your config file. Windows: Win + R → paste %APPDATA%\Claude\claude_desktop_config.json. macOS: open ~/Library/Application Support/Claude/claude_desktop_config.json.
  2. Add your ZeroCarbon bridge. Paste the block below and drop in your live API key.
  3. Restart Claude Desktop. A small plug icon appears in the chat input once it's connected.
{ "mcpServers": { "zerocarbon": { "command": "npx", "args": [ "-y", "@modelcontextprotocol/server-http-bridge", "--url", "https://zerocarbon.org.in/api/v1/mcp" ], "env": { "ZEROCARBON_API_KEY": "YOUR_zc_live_API_KEY_HERE" } } } }
  1. In Cursor, open Settings → Features → MCP.
  2. Click + Add New MCP Server.
  3. Set Name to ZeroCarbon and Type to command, then paste the command below.
  4. Add an env variable: key ZEROCARBON_API_KEY, value your live key. Save.
# Command npx -y @modelcontextprotocol/server-http-bridge --url \ https://zerocarbon.org.in/api/v1/mcp # Env ZEROCARBON_API_KEY=zc_live_...
  1. Add your inbox credentials to the environment so ZeroCarbon can read incoming invoices.
  2. Set a cron schedule on your hosting platform to scan the inbox automatically.
# Step 1 — environment EMAIL_USER="your-inbox-address@yourcompany.com" EMAIL_PASSWORD="your-secure-imap-app-password" EMAIL_HOST="imap.gmail.com" EMAIL_PORT=993 ZEROCARBON_API_KEY="zc_live_..." # Step 2 — cron scheduler { "crons": [ { "path": "/api/v1/company/automation/email-scan", "schedule": "0 0 * * *" } ] }
